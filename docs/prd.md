## 1. 产品概述

### 1.1 产品定位

一款帮助用户每天体验不同人格的趣味小程序，通过角色扮演的方式让用户突破自我、增添生活乐趣。

### 1.2 产品愿景

让每个人都能安全地尝试不同的生活方式，发现未知的自己。

### 1.3 核心价值主张

| 价值维度 | 说明 | 用户收益 |
|---------|------|----------|
| **趣味性** | 每天抽取不同人格 | 生活不无聊，充满期待 |
| **安全性** | 虚拟人格保护 | 在安全边界内尝试平时不敢做的事 |
| **成长性** | 扮演拓展性格边界 | 发现潜在能力，突破自我限制 |

### 1.4 目标用户画像

#### 核心用户群体

| 用户类型 | 特征描述 | 核心痛点 | 产品需求 |
|---------|---------|---------|---------|
| **社恐人群** | 想突破但不敢主动社交 | 缺乏尝试的勇气和理由 | 借人格名义尝试社交行为 |
| **生活无聊党** | 日常重复乏味 | 缺乏新鲜感和乐趣 | 寻找生活中的惊喜和话题 |
| **性格探索者** | 想了解不同的活法 | 没有安全的试错环境 | 安全体验不同人格 |
| **内容创作者** | 需要创作素材 | 灵感枯竭 | 获取灵感和创作话题 |

### 1.5 使用场景

- **通勤路上**：抽取今日人格，为一天设定角色
- **工作间隙**：查看人格任务，调整行为方式
- **睡前复盘**：打卡记录扮演情况，反思收获
- **社交分享**：分享人格卡片到朋友圈，引发讨论

## 2. 功能需求

### 2.1 MVP 功能（P0）

#### 2.1.1 抽取人格

**功能描述**：用户点击按钮随机获得今日人格

**用户故事**：
> 作为用户，我希望每天都能抽取到一个新的人格，这样我的一天就有了新的期待和玩法。

**交互流程**：
```
1. 用户进入首页，看到「抽取今日人格」按钮
2. 点击按钮后出现翻牌/抽卡动画（持续 2-3 秒）
3. 动画结束后展示抽中的人格卡片
4. 用户可查看详情或重新抽取
```

**业务规则**：

| 规则编号 | 规则说明 | 优先级 |
|---------|---------|-------|
| BR-001 | 每天只能抽取 1 次（凌晨 0 点刷新） | P0 |
| BR-002 | 已抽取的人格可重新抽取，消耗「重置币」 | P0 |
| BR-003 | 每日免费重置币 1 次 | P0 |
| BR-004 | 抽取结果保证 30 天内不重复 | P0 |

**验收标准**：
- [ ] 未抽取时显示抽取按钮
- [ ] 已抽取时显示今日人格卡片
- [ ] 动画流畅，时长 2-3 秒
- [ ] 凌晨 0 点正确刷新抽取次数

---

#### 2.1.2 人格卡片展示

**功能描述**：展示抽取到的人格详细信息

**卡片内容结构**：

| 模块 | 字段 | 说明 | 示例 |
|-----|------|------|------|
| 基础信息 | 人格名称 | 🎭 人格标识 | 冷酷杀手 |
| 基础信息 | 核心特点 | 📌 1-2 句话描述 | 话少、果断、不拖泥带水 |
| 行为指导 | 今日任务 | ✅ 3 个具体行动 | 说话不超过 10 个字/句 |
| 行为指导 | 说话方式 | 💬 对比示例 | ❌「我觉得可能 maybe 大概」→ ✅「行。」 |
| 外观建议 | 穿搭建议 | 👕 服饰风格（可选） | 黑/灰/深蓝色系 |
| 行为约束 | 今日禁忌 | ⚠️ 不要做的事 | 不要解释、不要道歉、不要犹豫 |

**交互功能**：

| 交互类型 | 说明 | 优先级 |
|---------|------|-------|
| 滑动浏览 | 可滑动查看完整内容 | P0 |
| 分享 | 点击生成分享海报 | P0 |
| 打卡 | 点击记录完成情况 | P0 |
| 返回 | 返回首页或上一级 | P0 |

**验收标准**：
- [ ] 所有字段正确显示
- [ ] 滑动流畅无卡顿
- [ ] 分享/打卡按钮可正常响应

---

#### 2.1.3 打卡记录

**功能描述**：用户记录当天扮演情况

**用户故事**：
> 作为用户，我希望记录自己的扮演情况，以便日后回顾和反思。

**打卡内容**：

| 字段 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| 评分 | 1-5 星 | 是 | 对今天扮演的满意度 |
| 一句话感受 | 文本 | 否 | 最多 100 字 |
| 照片 | 图片 | 否 | 可选上传 |

**业务规则**：

| 规则编号 | 规则说明 |
|---------|---------|
| BR-010 | 打卡时间：当天 00:00-23:59 有效 |
| BR-011 | 逾期不可补打卡 |
| BR-012 | 打卡后可修改，但仅限当天 |

**验收标准**：
- [ ] 评分必选
- [ ] 感受和照片可选
- [ ] 超时后打卡入口关闭

## 3. 人格库设计

### 3.1 人格数据结构

```yaml
id: personality_001                    # 唯一标识
name: 冷酷杀手                          # 人格名称
category: 性格系                        # 分类
difficulty: 3                           # 难度等级 (1-5)
description: 话少、果断、不拖泥带水       # 核心描述
tasks:                                  # 今日任务（3 个）
  - 说话不超过 10 个字/句
  - 做决定不超过 30 秒
  - 拒绝一个不必要的请求
speech_do:                              # 推荐说话方式
  - "行。"
  - "不行。"
  - "晚点说。"
speech_dont:                            # 避免说话方式
  - "我觉得可能 maybe 大概可以试试..."
outfit: 黑/灰/深蓝色系                   # 穿搭建议
taboo: 不要解释、不要道歉、不要犹豫        # 今日禁忌
unlock_type: free                       # 解锁类型 (free/premium)
```

## 9. 运营策略
